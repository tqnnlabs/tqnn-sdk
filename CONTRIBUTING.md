# Contributing

Thank you for your interest in contributing to the TQNN Python SDK.

The SDK provides the official Python interface for the TQNN Fault-Tolerant Inference Platform. Contributions that improve usability, reliability, documentation, and developer experience are always welcome.

---

# Ways to Contribute

You can contribute by:

- Reporting bugs
- Improving documentation
- Suggesting new features
- Improving code quality
- Expanding example applications
- Improving tests
- Reviewing pull requests

---

# Reporting Issues

When opening an issue, please include:

- Python version
- SDK version
- Operating system
- Steps to reproduce
- Expected behavior
- Actual behavior
- Error messages (if applicable)

Providing a minimal reproducible example helps us investigate issues more efficiently.

---

# Pull Requests

Before submitting a pull request:

- Ensure the project builds successfully.
- Keep changes focused on a single feature or fix.
- Follow the existing code style.
- Update documentation when behavior changes.
- Test your changes before submitting.

---

# Development Principles

The SDK is designed to be:

- Simple
- Lightweight
- Well documented
- Stable
- Consistent with the TQNN API

Contributions should preserve these principles whenever possible.

---

# Documentation

Documentation improvements are encouraged.

Examples include:

- Better examples
- Clearer explanations
- Typographical corrections
- Additional tutorials
- Improved API references

---

# Code Style

Please strive for:

- Readable code
- Meaningful variable names
- Consistent formatting
- Clear documentation
- Backward compatibility whenever practical

---

# Security

If you discover a potential security issue, please do not disclose it publicly.

Instead, contact TQNN Labs directly so the issue can be investigated and resolved responsibly.

**Contact:**

tqnnlabs@gmail.com

---

# Questions

Questions about the SDK or API are welcome through GitHub Discussions or GitHub Issues.

If your question relates to the hosted TQNN API, please include:

- SDK version
- API version
- A minimal code example (when applicable)
