# Versioning Policy

The SDK follows semantic versioning whenever practical.

- **Major** versions introduce breaking API compatibility changes.
- **Minor** versions introduce new functionality while maintaining compatibility.
- **Patch** versions include bug fixes, documentation improvements, and maintenance updates.
